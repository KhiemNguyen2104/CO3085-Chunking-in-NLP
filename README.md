# Chunking Techniques in Long Documents Summarization and Q&A

This project is an assignment that aims to research about the chunking concept in Natural Language Processing (NLP). The processes of problem specification, material exploration, research, and experiment conducting are presented in this GitHub repository.

## Repository Branches

Each branch in this repo is responsible for a task in this assignment, including:

1. `main`: The main branch that stores the timeline of the project and this description.
2. `report`: Store the PDF report of the project.
3. Others: Branches that present the research and experiment results for related tasks.

## Project Description

### Introduction

Summarization and Q&A are two major tasks that are the motivation of the NLP evolution throughout decades, starting from short and simple documents to long and complex ones nowadays. Although the achievements in machine learning have allowed models process and analyze larger documents, but processing a whole large document such as a legislation, a contract, etc. lead to limitations about time and resources. In many situations, our targets are summarizing the main points, or finding the answer from specific paragraphs, that do not require a fully processing for the document. Therefore, chunking has become a solution for efficiently handling long documents by breaking them into smaller, manageable segments. This approach not only helps in overcoming the context window constraints of models, but also improves retrieval accuracy and computational efficiency in downstream tasks. In this assignment, we will explore about the history of chunking techniques in NLP as well as conduct some experiments of state-of-the-art (SOTA) methods for benchmarking.
